# ページテーブル（ページリンク整理ボード）要件定義

> NotionPlus に「ページリンクを見出し付きの格子で整理する」ブロックを追加する。

> **デザイン改訂（コールアウト/Trello風カンバン化）= 下記 §A・実装済（2026-06-16）**

## A. デザイン改訂：コールアウト風カードへ（要件定義・未実装）

### A-1. 目的
- 現状は枠線つきの**表（テーブル）**に見え、「データ表」感が強く、気軽に使いづらい。
- Notion の**コールアウト**のような、**やわらかい角丸カード＋アイコン＋見出し＋字下げリスト**に寄せ、
  「整理メモ」として気持ちよく使える見た目にする。**テーブルを想起させない**のがゴール。

### A-2. 見た目（目標イメージ）
```
　┌──────────────────────┐   ┌──────────────────────┐
　│ 💡 見出し              │   │ 📚 教材               │
　│    ・ ページ1          │   │    ・ ページA          │
　│    ・ ページ2          │   │    ・ ページB          │
　│    ・ ページ3          │   │    ＋ 追加             │
　│    ＋ 追加             │   └──────────────────────┘
　└──────────────────────┘
```
- **1グループ＝1コールアウトカード**（現在の「列（小見出し＋リンク）」をカード化）。
- カード：**角丸（rounded-xl）＋淡い背景色＋内側パディング・枠線なし（境界線で区切らない）**。
- カード頭：**アイコン（絵文字・変更可）＋見出し（小見出し）**。例「💡 見出し」。
- カード中身：ページリンクを**字下げした箇条リスト**で表示（セル境界なし・ホバーで淡く反応）。
- **大見出し（セクション）**：カード群の上に**小さなラベル見出し**として置く（表のヘッダー帯はやめる）。
- レイアウト：カードを**横並び＋折り返し（flex-wrap, 最小幅 ~220px）**。画面が狭ければ自動で縦積み。

### A-3. データモデル（既存を拡張・後方互換）
既存 `PtColumn` に2フィールド追加（無ければ既定値で表示）：
```ts
interface PtColumn {
  id: string;
  heading: string;
  icon?: string;   // 追加: カードのアイコン（既定 '💡'）
  color?: string;  // 追加: カード背景色（既定 'グレー'=#F1F1EF、CALLOUT_BG_COLORS から選択）
  links: PtLink[];
}
```
- `sections`/`links` 構造は不変＝**既存のページテーブルもそのまま新デザインで表示**（移行不要）。
- 色は既存 `CALLOUT_BG_COLORS`（黄/青/緑/赤/紫/グレー）を流用。

### A-4. 操作（既存のまま＋カード化に伴う追加）
- 既存：見出し編集 / セクション・列(カード)の追加削除並べ替え / リンク追加(既存検索＋新規)・削除・上下移動・切り取り→貼り付け / クリック遷移。これらは**そのまま維持**。
- 追加：
  - カードの**アイコン変更**（クリックで絵文字ピッカー。ページリンクのアイコン選択を流用）。
  - カードの**背景色変更**（既存コールアウトの色ドット選択を流用）。

### A-5. 表示切替（任意）
- 既定を**カード表示**にする。「今のテーブルも良い」とのことなので、**表示モード切替（カード / コンパクト表）**を持たせるか要検討（§A-7 質問）。

### A-6. スコープ／段階
- **v1（最小）**：列をコールアウトカード化（角丸・淡色・アイコン＋見出し・字下げリスト・折り返し）。アイコン＆色の変更。既存操作は維持。
- **v2**：カードのドラッグ並べ替え・カード間ドラッグ移動、未配置の子ページ自動収集（既存 Phase 3 と合流）。

### A-7. 決定事項（2026-06-16・実装する）
1. **テーブル表示は残さない＝カード一本化**（A）。
2. **ヘッダーの装飾アイコンは無し**（💡等は付けない）。※ページリンク側の絵文字はページの識別子なので残す。
3. カード（リスト）の**既定背景色はグレー**（`#F1F1EF`）。色は各リストごとに変更可（既存6色）。
4. **大見出し（セクション）は残す**（カード群の上の小ラベルとして）。

### A-8. 参照イメージ＝Trello のカンバンビュー
- **列＝リスト**（小見出しヘッダー＋淡グレーの角丸コンテナ）。
- **各ページリンク＝白いカード**（角丸・薄い影・ページ名／ページ絵文字つき・クリックで遷移）。
- 各リスト下部に「**＋ カードを追加**」（Trello の "Add another card" 風）。
- リストは横並び＋折り返し。大見出しでまとめる。
- 方針：「**とりあえず作って使いながら改善**」。v1 を出して反応を見て詰める。

### A-9. データモデル（最終）
`PtColumn` に `color?: string`（既定 `#F1F1EF`）のみ追加（アイコンフィールドは持たない）。`sections`/`links` は不変＝既存ボードはそのまま新デザインで表示。

## 1. 目的・背景

- 子ページ（ノート・ブック）が増えると、サイドバーや本文中のリンクが**フラットな縦一列**になり、関連でまとめて見渡せない。
- これを**大見出し＋列（小見出し）＋各列に縦並びのページリンク**という格子に手で整理できるようにする。
- 位置づけは「**目次ボード / MOC（Map of Content）**」。データベースのような型付きの表ではなく、**リンクを並べて分類する見た目重視の整理ブロック**。

### 何ではないか（既存機能との切り分け）
| 機能 | 役割 | 本機能との違い |
|---|---|---|
| データベース（ページタイプ） | 型付き列＋行セル＋集計の表 | こちらは行＝データ。ページテーブルは**セル＝ページへのリンク** |
| インラインDB埋め込み | 別DBの中身をプレビュー | ページテーブルは**自分で手配置**して整理する |
| 目次（TOC）ノード | 1ページ内の見出しから自動生成 | ページテーブルは**複数ページのリンク**を束ねる |

→ **既存を置き換えず、新しい「ブロック（TipTapノード）」として追加**する。

## 2. コンセプト構造（見た目）

```
┌─────────────────────────────────────────────┐
│                大見出し（任意・列をまたぐ）              │
├───────────┬───────────┬───────────┬───────────┤
│  小見出し   │  小見出し   │  小見出し   │  小見出し   │  ← 列ヘッダー
├───────────┼───────────┼───────────┼───────────┤
│ 📄 ページ1  │ 📄 ページ1  │ 📄 ページ1  │ 📄 ページ1  │
│ 📄 ページ2  │ 📄 ページ2  │ 📄 ページ2  │ 📄 ページ2  │  ← 各列に
│ 📄 ページ3  │ 📄 ページ3  │ 📄 ページ3  │ 📄 ページ3  │     リンクを縦並び
│ ＋追加      │ ＋追加      │ ＋追加      │ ＋追加      │
└───────────┴───────────┴───────────┴───────────┘
```

- **大見出し**：ボード全体のタイトル（任意・空なら非表示）。
- **列（カラム）**：それぞれに**小見出し**を持つ。列数は可変（目安 1〜6）。
- **セル（リンク項目）**：各列は**ページリンクの縦リスト**。リンクはアイコン＋ページ名で表示し、クリックで遷移。
- 1ページ内に**複数のページテーブルを置ける**（章ごとに別ボード等）。

## 3. 実装方針

- **TipTap カスタムノード `pageTable`** として実装（既存の `pageLink` / `inlineDatabase` / `toc` ノードと同じ NodeView 方式）。
- スラッシュコマンド **`/ページテーブル`**（または `/整理ボード`）で挿入。
- データは**ページ本文の TipTap JSON 内（ノードの attrs）**に保持。**新規 Firestore コレクションは不要**。
- リンクは既存の**ページリンク概念**（href=`/notion-plus/{id}`）を流用。表示するアイコン・ページ名は `notionPageStore` から**ライブ取得**（リネーム・アイコン変更に追従）。

### データ構造（ノード attrs 案）
```ts
interface PageTableData {
  title: string;                  // 大見出し（空可）
  columns: {
    id: string;
    heading: string;              // 小見出し
    links: { href: string; title: string; icon: string }[];
  }[];
}
```
※ `title`/`icon` はフォールバック用スナップショット。表示はストアのライブ値を優先（既存 PageLinkView と同方針）。

### 親子関係（ツリー追従）
- ページテーブルに置いたサブページリンクは、既存の **`reconcileChildrenParent`** によって**このページの子**として `parentId` が追従する（サイドバーのツリーと整合）。
- ＝「リンクを整理ボードに置く＝子として束ねる」が自然に成立。

## 4. 主要操作

### Phase 1（MVP）
- [ ] `/ページテーブル` で挿入（初期＝大見出し空・2列・各列空）
- [ ] 大見出し / 小見出しの**インライン編集**（クリックで編集）
- [ ] 列に**リンクを追加**：「＋追加」→ **ページピッカー**（既存ページを検索して選ぶ）／**新規サブページ作成**して追加
- [ ] リンクを**削除**（各項目のホバー × ／ 右クリック）
- [ ] リンク**クリックで遷移**（モーダル内は `PageNavigationContext` 経由）
- [ ] **列の追加 / 削除**
- [ ] ライブのアイコン・ページ名表示（リネーム追従）

### Phase 2（整理しやすく）
- [ ] 列内リンクの**並べ替え**（↑↓ or ドラッグ）
- [ ] **列の並べ替え**
- [ ] リンクを**列間でドラッグ移動**
- [ ] **複数の大見出しセクション**（or ボード複製）

### Phase 3（便利機能）
- [ ] **未配置の子ページを自動収集**（「このページの子で、まだボードに無いもの」をワンクリックで流し込み）
- [ ] 表示オプション：アイコン表示 ON/OFF、列幅、コンパクト表示、列の色／背景
- [ ] リンクごとの**ひとことメモ**（補足テキスト）
- [ ] 空リンク切れ（削除済みページ）の**自動グレーアウト**（既存 `notion-child://` 未解決表示と同方針）

## 6. 非機能・地雷メモ
- **本文非破壊の原則**：ノード attrs に閉じる。既存ページの内容には影響させない。
- **モバイル（React Native）**：Web の TipTap ノードはそのまま出ないため、Android では**読み取り表示**を最低限担保（編集は Web/デスクトップ優先）。要・別途確認。
- **パフォーマンス**：1ボードのリンク数が多い場合でも、表示はストア参照のみ（重い購読を増やさない）。
- **後方互換**：`pageTable` 未対応バージョンで開いても本文が壊れないよう、`parseHTML`/`renderHTML` を素直な `div[data-type="page-table"]` にする。
