# ページテーブル（ページリンク整理ボード）要件定義

> NotionPlus に「ページリンクを見出し付きの格子で整理する」ブロックを追加する。
> 作成日: 2026-06-16 ／ ステータス: **Phase 1＋2 実装済（2026-06-16）**・Phase 3 未着手

---

## 1. 目的・背景

- 子ページ（ノート・ブック）が増えると、サイドバーや本文中のリンクが**フラットな縦一列**になり、関連でまとめて見渡せない。
- これを**大見出し＋列（小見出し）＋各列に縦並びのページリンク**という格子に手で整理できるようにする。
- 位置づけは「**目次ボード / MOC（Map of Content）**」。データベースのような型付きの表ではなく、**リンクを並べて分類する見た目重視の整理ブロック**。

### 何ではないか（既存機能との切り分け）
| 機能 | 役割 | 本機能との違い |
|---|---|---|
| データベース（ページタイプ） | 型付き列＋行セル＋集計の表 | こちらは行＝データ。ページテーブルは**セル＝ページへのリンク** |
| インラインDB埋め込み | 別DBの中身をプレビュー | ページテーブルは**自分で手配置**して整理する |
| 目次（TOC）ノード | 1ページ内の見出しから自動生成 | ページテーブルは**複数ページのリンク**を束ねる |

→ **既存を置き換えず、新しい「ブロック（TipTapノード）」として追加**する。

## 2. コンセプト構造（見た目）

```
┌─────────────────────────────────────────────┐
│                大見出し（任意・列をまたぐ）              │
├───────────┬───────────┬───────────┬───────────┤
│  小見出し   │  小見出し   │  小見出し   │  小見出し   │  ← 列ヘッダー
├───────────┼───────────┼───────────┼───────────┤
│ 📄 ページ1  │ 📄 ページ1  │ 📄 ページ1  │ 📄 ページ1  │
│ 📄 ページ2  │ 📄 ページ2  │ 📄 ページ2  │ 📄 ページ2  │  ← 各列に
│ 📄 ページ3  │ 📄 ページ3  │ 📄 ページ3  │ 📄 ページ3  │     リンクを縦並び
│ ＋追加      │ ＋追加      │ ＋追加      │ ＋追加      │
└───────────┴───────────┴───────────┴───────────┘
```

- **大見出し**：ボード全体のタイトル（任意・空なら非表示）。
- **列（カラム）**：それぞれに**小見出し**を持つ。列数は可変（目安 1〜6）。
- **セル（リンク項目）**：各列は**ページリンクの縦リスト**。リンクはアイコン＋ページ名で表示し、クリックで遷移。
- 1ページ内に**複数のページテーブルを置ける**（章ごとに別ボード等）。

## 3. 実装方針

- **TipTap カスタムノード `pageTable`** として実装（既存の `pageLink` / `inlineDatabase` / `toc` ノードと同じ NodeView 方式）。
- スラッシュコマンド **`/ページテーブル`**（または `/整理ボード`）で挿入。
- データは**ページ本文の TipTap JSON 内（ノードの attrs）**に保持。**新規 Firestore コレクションは不要**。
- リンクは既存の**ページリンク概念**（href=`/notion-plus/{id}`）を流用。表示するアイコン・ページ名は `notionPageStore` から**ライブ取得**（リネーム・アイコン変更に追従）。

### データ構造（ノード attrs 案）
```ts
interface PageTableData {
  title: string;                  // 大見出し（空可）
  columns: {
    id: string;
    heading: string;              // 小見出し
    links: { href: string; title: string; icon: string }[];
  }[];
}
```
※ `title`/`icon` はフォールバック用スナップショット。表示はストアのライブ値を優先（既存 PageLinkView と同方針）。

### 親子関係（ツリー追従）
- ページテーブルに置いたサブページリンクは、既存の **`reconcileChildrenParent`** によって**このページの子**として `parentId` が追従する（サイドバーのツリーと整合）。
- ＝「リンクを整理ボードに置く＝子として束ねる」が自然に成立。

## 4. 主要操作

### Phase 1（MVP）
- [ ] `/ページテーブル` で挿入（初期＝大見出し空・2列・各列空）
- [ ] 大見出し / 小見出しの**インライン編集**（クリックで編集）
- [ ] 列に**リンクを追加**：「＋追加」→ **ページピッカー**（既存ページを検索して選ぶ）／**新規サブページ作成**して追加
- [ ] リンクを**削除**（各項目のホバー × ／ 右クリック）
- [ ] リンク**クリックで遷移**（モーダル内は `PageNavigationContext` 経由）
- [ ] **列の追加 / 削除**
- [ ] ライブのアイコン・ページ名表示（リネーム追従）

### Phase 2（整理しやすく）
- [ ] 列内リンクの**並べ替え**（↑↓ or ドラッグ）
- [ ] **列の並べ替え**
- [ ] リンクを**列間でドラッグ移動**
- [ ] **複数の大見出しセクション**（or ボード複製）

### Phase 3（便利機能）
- [ ] **未配置の子ページを自動収集**（「このページの子で、まだボードに無いもの」をワンクリックで流し込み）
- [ ] 表示オプション：アイコン表示 ON/OFF、列幅、コンパクト表示、列の色／背景
- [ ] リンクごとの**ひとことメモ**（補足テキスト）
- [ ] 空リンク切れ（削除済みページ）の**自動グレーアウト**（既存 `notion-child://` 未解決表示と同方針）

- 各リンク先ページの**学習ステータス／復習待ち**を小さなバッジで列に表示（learningItems 連携）。
- ＝「教材ボード」として、どの章が復習待ちかを一覧で把握できる。
- Phase 3 以降の検討事項（MVP では入れない）。

---

## 6. 非機能・地雷メモ
- **本文非破壊の原則**：ノード attrs に閉じる。既存ページの内容には影響させない。
- **モバイル（React Native）**：Web の TipTap ノードはそのまま出ないため、Android では**読み取り表示**を最低限担保（編集は Web/デスクトップ優先）。要・別途確認。
- **パフォーマンス**：1ボードのリンク数が多い場合でも、表示はストア参照のみ（重い購読を増やさない）。
- **後方互換**：`pageTable` 未対応バージョンで開いても本文が壊れないよう、`parseHTML`/`renderHTML` を素直な `div[data-type="page-table"]` にする。
